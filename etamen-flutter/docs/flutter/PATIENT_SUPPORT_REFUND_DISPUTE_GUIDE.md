# Patient Support, Refund, and Dispute Guide

This guide covers the local patient-facing support foundation.

## Support Tickets

1. Open Account.
2. Open support tickets.
3. Create a ticket with category, subject, and description.
4. Reply from ticket details when needed.
5. Patient can see only their own tickets.

Support statuses:

- `open`
- `pending_admin`
- `pending_user`
- `resolved`
- `closed`

## Refund Requests

1. Open refunds.
2. Create refund request with reason and context/payment when available.
3. Track request status.

Refund statuses:

- `requested`
- `under_review`
- `approved`
- `rejected`
- `processed`
- `cancelled`

Important:

- `approved` means platform decision only.
- `processed` means manual confirmation only.
- No live refund gateway exists in the local foundation.

## Disputes

1. Open disputes.
2. Create dispute with reason and context/payment when available.
3. Track status until resolved or closed.

Dispute statuses:

- `open`
- `investigating`
- `waiting_user`
- `waiting_provider`
- `resolved`
- `rejected`
- `closed`

## Safety and Privacy

- Support is not for emergencies.
- Etamen support does not diagnose or prescribe.
- If the case is urgent, use emergency services.
- Internal admin notes are not visible to patient users.
- Patient should not see another user's tickets, refunds, or disputes.
